## install

```
npm install -g nocli
```

## usage

```
$ nocli --help
Usage: nocli [options] module

Options:
  --help  Show help                                                    [boolean]
  -c      read options from yaml config file
```

## why?
I sometimes write small, once-off scripts that require some configuration. For these cases, I'd rather rely on something else to worry about reading in configuration and expose these scripts as functions taking in an options object.
